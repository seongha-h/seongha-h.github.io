# 문학 텍스트 단어 빈도 분석 사이트

사이트 주소: https://username.github.io

## 1. 사이트 소개

이 사이트는 문학 작품의 텍스트를 불러와 단어 빈도를 분석하고, 그 결과를 차트로 시각화하는 웹사이트이다.  
한 학기 동안 배운 HTML, Markdown, JavaScript, fetch, Chart.js 등을 활용하여 텍스트 분석 도구를 직접 구현하였다.

사이트는 크게 다음 네 부분으로 구성되어 있다.

- 홈
- 두 작품 비교 페이지
- 텍스트 분석기
- 구현 과정 블로그 글

## 2. 홈

홈 화면은 사이트의 전체 목적과 주요 기능을 소개하는 페이지이다.  
사용자는 홈에서 각 분석 페이지로 이동할 수 있으며, 사이트가 문학 텍스트의 단어 빈도를 분석하고 시각화하는 프로젝트임을 확인할 수 있다.

## 3. 두 작품 비교 페이지

두 작품 비교 페이지에서는 Project Gutenberg에서 가져온 문학 작품 두 편을 분석한다.  
예를 들어 『A Study in Scarlet』과 『The Hound of the Baskervilles』의 텍스트를 각각 불러온 뒤, 불용어를 제거하고 상위 단어 빈도를 계산한다.

분석 결과는 Chart.js를 이용해 막대그래프로 표시된다.  
이를 통해 사용자는 두 작품에서 어떤 단어가 자주 등장하는지 한눈에 비교할 수 있다.

## 4. 분석기

분석기 페이지에서는 사용자가 직접 텍스트를 입력할 수 있다.  
입력된 텍스트는 JavaScript를 통해 소문자로 변환되고, 구두점이 제거된 뒤 단어 단위로 나뉜다.  
이후 불용어 목록을 적용하여 의미가 적은 단어를 제거하고, 남은 단어들의 빈도를 계산한다.

분석 결과는 상위 20개 단어를 기준으로 막대그래프로 나타난다.  
또한 이미 그려진 차트가 있을 경우 `destroy()`를 사용하여 기존 차트를 지우고 새로운 결과를 다시 그리도록 구현하였다.

## 5. 블로그

블로그 글에는 사이트를 구현하는 과정에서 겪은 문제와 해결 과정, 그리고 분석 결과에서 발견한 점을 정리하였다.  
특히 JavaScript 파일을 여러 개로 나누어 관리할 때 발생한 `<script>` 로드 순서 문제, `fetch` 경로 문제, 차트 갱신 문제 등을 다루었다.

또한 단어 빈도 분석을 통해 문학 작품의 분위기나 주제와 관련된 어휘가 반복적으로 나타난다는 점을 확인하고, 데이터 분석이 문학 작품을 이해하는 또 다른 방법이 될 수 있음을 정리하였다.
